    Function: chunk_text(text: str, max_tokens: int = 400, overlap_tokens: int = 50) -> list[str]
    - Use tiktoken.get_encoding("cl100k_base") at module level (matches text-embedding-3-small)
    - Split text on "\n\n" into paragraphs
    - Accumulate paragraphs into buffer; when adding next paragraph would exceed max_tokens, finalize chunk
    - Overlap: backfill final paragraphs from previous buffer until overlap_tokens budget used
    - Edge cases: empty/whitespace → [], single oversized paragraph → return as-is (soft limit)

STEP 3 — LangGraph graph  [commit: "Phase 2.8+9: knowledge agent, supervisor, LangGraph graph"]

  backend/agents/graph.py:
    AgentState (TypedDict):
      messages: list[dict]        # [{"role": "customer"|"agent", "content": str}]
      customer_id: str
      customer_context: dict      # Phase 2: always {}; Phase 3 will populate from DB
      current_intent: str         # set by supervisor
      routing_decision: str       # set by supervisor (for audit logging)
      confidence: float           # set by knowledge agent
      response: str               # final response text
      requires_escalation: bool   # Phase 3: set when confidence < threshold
      actions_taken: list[dict]   # audit trail

    build_graph() → compiled graph:
      builder = StateGraph(AgentState)
      builder.add_node("supervisor", supervisor_node)
      builder.add_node("knowledge_agent", knowledge_agent_node)
      builder.add_edge(START, "supervisor")
      builder.add_conditional_edges("supervisor", route_after_supervisor, {"knowledge_agent": "knowledge_agent"})
      builder.add_edge("knowledge_agent", END)
      return builder.compile()

    graph = build_graph()  ← module-level, reused across all requests

STEP 4 — Knowledge agent  [same commit as step 3]

  backend/agents/knowledge_agent.py:
    async def knowledge_agent_node(state: AgentState, config: RunnableConfig) -> dict:
      1. Extract query: last message where role == "customer"
      2. Embed: litellm.aembedding(model=settings.litellm_embedding_model, input=[query])
         → query_embedding = response["data"][0]["embedding"]
      3. Vector search via raw SQL (pass embedding as str(list) — pgvector accepts '[0.1, 0.2, ...]'):
           SELECT c.id, c.chunk_text, d.title, d.category,
                  (c.embedding <=> :embedding) AS cosine_distance
           FROM kb_chunks c JOIN kb_documents d ON c.document_id = d.id
           WHERE c.embedding IS NOT NULL
           ORDER BY c.embedding <=> :embedding LIMIT 5
         DB session from: db = config["configurable"]["db"]
      4. Format context: join chunks with "\n\n---\n\n", include source title
      5. Build messages for LLM:
           system: SYSTEM_PROMPT.format(context=context)  ← constant at top of file
           then: last max_context_messages from state["messages"]
      6. litellm.acompletion(model=settings.litellm_model, messages=..., stream=False)
         → agent_response = response.choices[0].message.content
      7. confidence = 1.0 - chunks[0].cosine_distance  (0=identical, 1=orthogonal)
      8. Return: {"response": agent_response, "confidence": confidence, "actions_taken": [...]}

    IMPORTANT: Do NOT commit inside the node. The endpoint owns the commit.


STEP 5 — SSE streaming endpoint  [commit: "Phase 2.10: SSE streaming endpoint wired to LangGraph"]

  backend/routers/chat.py — replace 501 stub with:

    @router.get("/chat/stream/{conversation_id}")
    async def stream_response(conversation_id: str, db: AsyncSession = Depends(get_db)):
      1. Validate conversation exists (404 if not)
      2. Load latest customer message (422 if none)
      3. Load full conversation history (up to max_context_messages), build messages_for_state
      4. Build initial_state dict (all AgentState keys, customer_context={})
      5. Define async generator event_generator():
           config = {"configurable": {"db": db}}
           async for chunk in graph.astream(initial_state, config=config, stream_mode="updates"):
               if "knowledge_agent" in chunk:
                   final_output = chunk["knowledge_agent"]
           # Stream response word-by-word
           for word in final_output["response"].split(" "):
               yield ServerSentEvent(data=word + " ", event="token")
           # Persist to DB
           agent_msg = Message(id=uuid4(), conversation_id=..., role="agent",
                               content=full_response, agent_type="knowledge")
           db.add(agent_msg); await db.flush()
           db.add(AuditLog(id=uuid4(), conversation_id=..., message_id=agent_msg.id,
                           agent_type="knowledge", action="search_kb",
                           routing_decision=..., confidence=...))
           await db.commit()
           yield ServerSentEvent(data="", event="done")
      6. Return EventSourceResponse(event_generator())

    NOTE: Using graph.astream(stream_mode="updates") + word-by-word emission rather than
    astream_events, because litellm.acompletion is called directly (not via LangChain Runnable)
    so on_chat_model_stream events won't fire. Word-by-word is sufficient for Phase 2.
