Request Execution Paths

Knowledge Query — "What is your return policy?"
POST /api/chat → GET /api/chat/stream → load history + customer_context → graph START → conversation_agent Pass 1 → input_guard ✓ → LLM → knowledge_query (0.95) → pending_service="knowledge" → knowledge_service → embed query → pgvector search (5 chunks) → conversation_agent Pass 2 → similarity ≥ 0.7 ✓ → LLM + KB context → response → output_guard ✓ → END → stream tokens → persist Message + AuditLog
Action Request — "Cancel my order"
conversation_agent Pass 1 → LLM → action_request (cancel_order) → pending_service="action" → pending_action={tool:"cancel_order", params:{}} → action_service → strip null params → TOOL_REGISTRY["cancel_order"] → validate ownership → DB: set cancelled → conversation_agent Pass 2 → LLM + action result → response → output_guard ✓ → END
Escalation — "I want to speak to a human" or low KB confidence
conversation_agent Pass 1 → LLM → escalation_request OR Pass 2: similarity < 0.7 OR output_guard fails → pending_service="escalation" → escalation_handler → write Escalation row → conversations.status="escalated" → handoff message (by reason) → END → stream handoff
Blocked Input — prompt injection or abusive content
conversation_agent Pass 1 → input_guard Stage 1: regex match → OR Stage 2: LLM → prompt_injection / abusive / off_topic → return blocked_response → pending_service="" → END → stream safe reply — no escalation
